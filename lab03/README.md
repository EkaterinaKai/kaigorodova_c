# Лабораторная работа 3: Динамические массивы в языке C
## Цель лабораторной работы
Изучить работу с динамической памятью в языке C: освоить функции `malloc`, `calloc`, `realloc`, `free`, научиться создавать, изменять и удалять динамические массивы, а также правильно управлять памятью для предотвращения утечек и других ошибок.
## Основные задачи:
### Базовые операции с динамической памятью
- Выделение памяти с помощью `malloc`
- Выделение и обнуление памяти с помощью `calloc`
- Изменение размера выделенной памяти через `realloc`
- Освобождение памяти через `free`
### Динамические массивы
- Создание динамических массивов различных типов
- Заполнение и вывод динамических массивов
- Поиск элементов (максимум, минимум, сумма)
### Модификация динамических массивов
- Увеличение размера массива
- Уменьшение размера массива
- Удаление элемента по индексу
- Вставка элемента в заданную позицию
### Динамические массивы структур
- Выделение памяти под массив структур
- Заполнение и обработка массива структур
- Поиск по полям структуры
### Комплексное управление данными
- Создание динамического списка с функциями добавления, удаления, вывода
- Реализация меню для управления списком
## Используемые технологии и библиотеки
**Язык программирования:** C

## Ответы на контрольные вопросы:
1. Что делает malloc
Выделяет блок памяти заданного размера в байтах. Не инициализирует память.
2. Чем отличается malloc от calloc
calloc выделяет память под count элементов по size байт и обнуляет её.
malloc только выделяет (мусор).
3. Что делает realloc
Изменяет размер ранее выделенного блока памяти. Сохраняет данные в пределах нового размера.
4. Что произойдет при утечке памяти
Программа потребляет всё больше памяти, что может привести к замедлению или аварийному завершению (крах ОС/программы).
5. Зачем нужен free
Возвращает память системе. Иначе — утечка.
6. Что произойдет при двойном освобождении памяти
Неопределенное поведение (UB). Часто — сбой программы.
7. Как проверить, что malloc вернул NULL
if (ptr == NULL) { /* ошибка */ }
8. Отличие статического и динамического массива
Статический размер фиксирован на этапе компиляции, память выделяется автоматически.
Динамический размер задаётся во время выполнения, память управляется вручную.
